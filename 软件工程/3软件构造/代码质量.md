## 代码质量常见问题总览

常见问题可以概括为：

| 问题类型     | 典型表现                    | 修改方向         |
| -------- | ----------------------- | ------------ |
| 命名不清晰    | `a`、`b`、`data` 含义不明     | 使用能表达业务含义的名称 |
| 函数过长     | 一个函数做很多事                | 拆分函数，单一职责    |
| 重复代码     | 多处复制相同逻辑                | 提取函数、提取类     |
| 魔法值      | 代码中直接写 `60`、`100`、`"A"` | 提取常量         |
| 过度使用全局变量 | 多处共享可变状态                | 封装状态，显式传参    |
| 高耦合      | 一个类直接依赖很多具体类            | 面向接口编程       |
| 低内聚      | 一个类承担多种职责               | 拆分类，职责分离     |
| 复杂条件判断   | 多层 `if-else`            | 提前返回、策略模式、多态 |
| 异常处理不当   | 吞掉异常或乱抛异常               | 明确错误语义       |
| 注释质量差    | 注释重复代码或与代码不一致           | 用清晰命名代替无效注释  |
