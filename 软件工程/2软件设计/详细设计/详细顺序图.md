## 详细顺序图

#### 参与者 / 对象

> 参与者或对象放在顺序图顶部，表示参与交互的实体。

图例：

```text
┌──────────┐      ┌────────────┐
│  用户     │      │  系统对象    │
└──────────┘      └────────────┘
```

常见写法：

```text
用户
loginUI:LoginView
service:UserService
dao:UserDAO
```

#### 生命线 Lifeline

> 生命线表示对象在交互过程中的存在时间，通常画成对象下面的一条竖直虚线。

图例：

```text
┌──────────┐
│  对象A    │
└──────────┘
     │
     │
     │
     │
```

#### 激活条 Activation

> 激活条表示对象正在执行某个操作，通常画在生命线上，是一个细长矩形。

图例：

```text
对象A              对象B
 │                 │
 │  call()         │
 ├───────────────▶ │
 │                 █
 │                 █
 │                 █
 │  return         │
 │ < - - - - - - -┤
 │                 │
```

#### 同步消息 Synchronous Message

> 同步消息表示调用方发出调用后要等待接收方完成，常用于普通方法调用、接口调用、数据库查询等。
> 用**实线 + 实心三角箭头**表示

图例：

```text
对象A              对象B
 │                 │
 │  同步调用        │
 ├───────────────▶ │
 │                 │
```

#### 异步消息 Asynchronous Message

> 异步消息表示发送方发出消息后不等待接收方完成，而是继续执行。
> 用**实线 + 开放箭头 / 鱼骨箭头**表示，而不是虚线。
图例：

```text
对象A              对象B
 │                 │
 │  异步消息        │
 ├───────────────▷ │
 │                 │
 │ 继续执行         │
 │                 │
```


> 通俗理解：异步消息就是“我通知你，不等你处理完”。

#### 返回消息 Reply / Return Message

> 返回消息表示被调用对象执行完成后把结果返回给调用对象。
> 用**虚线 + 开放箭头**表示
图例：

```text
对象A              对象B
 │                 │
 │  call()         │
 ├───────────────▶ │
 │  result         │
 │ < - - - - - - -┤
 │                 │
```


#### 创建消息 Create Message

> 创建消息表示一个对象在交互过程中创建另一个对象。被创建对象的生命线通常从创建消息到达的位置开始。

图例：

```text
对象A              对象B
 │                 ┌──────────┐
 │  «create»       │  对象B    │
 ├───────────────▶ └──────────┘
 │                    │
 │                    │
```

> 易错点：新对象的生命线不一定从图最上方开始，而是从被创建的位置开始。

#### 销毁消息 Delete / Destroy Message

> 销毁消息表示对象生命周期结束，通常在生命线末端画 `X`。

图例：

```text
对象A              对象B
 │                 │
 │  destroy()      │
 ├───────────────▶ │
 │                 X
```

#### 自调用消息 Self Message

> 自调用消息表示对象调用自身的方法。

图例：

```text
对象A
 │
 █───┐
 █   │ validate()
 █◀──┘
 │
```

例子：

```text
UserService : validatePassword()
```
> 易错点：不要为了自调用再画一个相同对象。

## 技巧
对于一个系统的详细顺序，可能拆分为多个业务，例如支付业务、积分业务，而不同业务是由不同对象完成的，例如支付逻辑对象，支付数据对象、积分逻辑对象、积分数据对象

一般而言，业务由系统发起，也就是总的逻辑对象，向每个业务所属逻辑对象发起业务请求，逻辑对象激活后处理业务，向数据对象请求、处理数据，将结果返回
