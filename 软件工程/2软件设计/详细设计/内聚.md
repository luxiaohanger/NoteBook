# 内聚

**内聚**指一个模块内部各个元素之间联系的紧密程度。内聚越高，说明模块内部职责越集中；内聚越低，说明模块内部功能越杂乱。

> 通俗理解：**内聚就是“一个模块自己内部有多专一”**。越专一越好，越杂越差。

> 考试常考排序：一般认为从较差到较好大致是：**偶然内聚 < 逻辑内聚 < 时间内聚 < 过程内聚 < 通信内聚 < 顺序内聚 < 功能内聚**。

---

## 偶然内聚

**偶然内聚**指模块中的功能没有明显关系，只是偶然被放在一起。

### 修改前：无关功能放在同一个类中

```java
class Utility {
    public void printReport() {
        System.out.println("打印报表");
    }

    public void calculateTax() {
        System.out.println("计算税费");
    }

    public void sendEmail() {
        System.out.println("发送邮件");
    }
}
```

这里 `printReport`、`calculateTax`、`sendEmail` 没有共同目标，只是随意放在 `Utility` 中。

### 修改后：按职责拆分

```java
class ReportPrinter {
    public void printReport() {
        System.out.println("打印报表");
    }
}

class TaxCalculator {
    public void calculateTax() {
        System.out.println("计算税费");
    }
}

class EmailService {
    public void sendEmail() {
        System.out.println("发送邮件");
    }
}
```

> 通俗理解：**把毫不相关的东西塞进同一个抽屉**。

> 修改方向：**按职责拆分类或模块，让一个模块只负责一类事情**。

## 逻辑内聚

**逻辑内聚**指模块中的功能在逻辑类别上相似，但具体执行哪个功能由参数决定。

### 修改前：用 type 参数选择功能

```java
class InputHandler {
    public void handleInput(String input, String type) {
        if ("keyboard".equals(type)) {
            System.out.println("处理键盘输入：" + input);
        } else if ("mouse".equals(type)) {
            System.out.println("处理鼠标输入：" + input);
        } else if ("file".equals(type)) {
            System.out.println("处理文件输入：" + input);
        }
    }
}
```

这些功能都属于“输入处理”，但具体处理逻辑不同，由 `type` 决定。

### 修改后：拆分为不同处理器

```java
interface InputHandler {
    void handle(String input);
}

class KeyboardInputHandler implements InputHandler {
    public void handle(String input) {
        System.out.println("处理键盘输入：" + input);
    }
}

class MouseInputHandler implements InputHandler {
    public void handle(String input) {
        System.out.println("处理鼠标输入：" + input);
    }
}

class FileInputHandler implements InputHandler {
    public void handle(String input) {
        System.out.println("处理文件输入：" + input);
    }
}
```

> 通俗理解：**看起来是一类事，但每次靠参数决定到底做哪一件**。

> 修改方向：**拆方法、拆类，或使用多态、策略模式替代类型判断**。

## 时间内聚

**时间内聚**指模块中的多个操作因为在同一时间段执行而被放在一起，例如系统启动、初始化、关闭时的一组操作。

### 修改前：把启动时所有事情塞在一起

```java
class SystemInitializer {
    public void init() {
        System.out.println("加载配置");
        System.out.println("连接数据库");
        System.out.println("清理临时文件");
        System.out.println("启动定时任务");
    }
}
```

这些操作共同点只是“启动时执行”，但职责并不完全相同。

### 修改后：拆分具体职责，由初始化流程统一调用

```java
class ConfigLoader {
    public void load() {
        System.out.println("加载配置");
    }
}

class DatabaseConnector {
    public void connect() {
        System.out.println("连接数据库");
    }
}

class TempFileCleaner {
    public void clean() {
        System.out.println("清理临时文件");
    }
}

class JobScheduler {
    public void start() {
        System.out.println("启动定时任务");
    }
}

class SystemInitializer {
    private ConfigLoader configLoader = new ConfigLoader();
    private DatabaseConnector databaseConnector = new DatabaseConnector();
    private TempFileCleaner tempFileCleaner = new TempFileCleaner();
    private JobScheduler jobScheduler = new JobScheduler();

    public void init() {
        configLoader.load();
        databaseConnector.connect();
        tempFileCleaner.clean();
        jobScheduler.start();
    }
}
```

> 通俗理解：**这些事不是因为目标相同，而只是因为同一时间发生**。

> 修改方向：**把具体职责拆出去，保留一个清晰的流程协调模块**。

---

## 过程内聚

**过程内聚**指模块中的多个操作按照某个执行顺序组织在一起，但这些操作处理的数据或目标不一定高度相关。

### 修改前：按流程顺序写在一起

```java
class OrderProcessor {
    public void process() {
        System.out.println("读取用户信息");
        System.out.println("检查库存");
        System.out.println("生成订单");
        System.out.println("发送通知");
    }
}
```

这些步骤有先后顺序，但每一步的职责差异较大。

### 修改后：流程控制与具体职责分离

```java
class UserReader {
    public void read() {
        System.out.println("读取用户信息");
    }
}

class InventoryChecker {
    public void check() {
        System.out.println("检查库存");
    }
}

class OrderCreator {
    public void create() {
        System.out.println("生成订单");
    }
}

class NotificationService {
    public void send() {
        System.out.println("发送通知");
    }
}

class OrderProcessor {
    private UserReader userReader = new UserReader();
    private InventoryChecker inventoryChecker = new InventoryChecker();
    private OrderCreator orderCreator = new OrderCreator();
    private NotificationService notificationService = new NotificationService();

    public void process() {
        userReader.read();
        inventoryChecker.check();
        orderCreator.create();
        notificationService.send();
    }
}
```

> 通俗理解：**只是因为执行顺序连在一起，不代表它们属于同一个职责**。

> 修改方向：**拆分职责，让流程模块只负责组织顺序，具体功能交给专门模块**。

## 通信内聚

**通信内聚**指模块中的多个操作使用相同的数据或作用于同一个数据对象。

### 示例：多个操作围绕同一个订单数据

```java
class OrderService {
    public void calculateTotal(Order order) {
        System.out.println("计算订单总价");
    }

    public void validateOrder(Order order) {
        System.out.println("校验订单");
    }

    public void saveOrder(Order order) {
        System.out.println("保存订单");
    }
}

class Order {
    int id;
    double totalPrice;
}
```

这些操作都围绕 `Order` 进行，因此比过程内聚更好。

### 进一步优化：区分领域逻辑和持久化逻辑

```java
class Order {
    int id;
    double totalPrice;

    public void calculateTotal() {
        System.out.println("计算订单总价");
    }

    public void validate() {
        System.out.println("校验订单");
    }
}

class OrderRepository {
    public void save(Order order) {
        System.out.println("保存订单");
    }
}
```

> 通俗理解：**大家都在处理同一个对象，所以联系比较紧密**。

> 修改方向：**继续按职责细分，把业务计算、校验、保存等不同职责适当拆开**。

## 顺序内聚

**顺序内聚**指模块中的多个操作前后相接，前一个操作的输出正好作为后一个操作的输入。

### 示例：数据处理流水线

```java
class DataProcessor {
    public void process(String rawData) {
        String cleanedData = clean(rawData);
        String parsedData = parse(cleanedData);
        save(parsedData);
    }

    private String clean(String rawData) {
        return "清洗后的数据";
    }

    private String parse(String cleanedData) {
        return "解析后的数据";
    }

    private void save(String parsedData) {
        System.out.println("保存：" + parsedData);
    }
}
```

这里 `clean` 的输出给 `parse`，`parse` 的输出给 `save`，步骤之间联系较强。

### 修改方向：保持清晰的数据流

```java
class DataProcessor {
    public void process(String rawData) {
        String cleanedData = clean(rawData);
        String parsedData = parse(cleanedData);
        save(parsedData);
    }

    private String clean(String rawData) {
        return rawData.trim();
    }

    private String parse(String cleanedData) {
        return cleanedData.toUpperCase();
    }

    private void save(String parsedData) {
        System.out.println("保存：" + parsedData);
    }
}
```

> 通俗理解：**上一道工序的产物正好交给下一道工序**。

> 修改方向：**保持输入输出关系清晰，必要时使用管道、流水线结构表达数据流**。

## 功能内聚

**功能内聚**指模块中的所有元素都为完成同一个明确功能服务，是最理想的内聚形式。

### 示例：专门负责密码加密

```java
class PasswordEncoder {
    public String encode(String rawPassword) {
        return "加密后的密码：" + rawPassword.hashCode();
    }
}
```

这个类只负责一件事：加密密码。

### 示例：专门负责订单总价计算

```java
class OrderPriceCalculator {
    public double calculate(double price, int count) {
        return price * count;
    }
}
```

> 通俗理解：**一个模块只干一件明确的事，并且把这件事干完整**。

> 修改方向：**保持单一职责，避免把无关功能塞进来**。

## 内聚类型对比

| 类型   | 典型表现               | 好坏程度 | 修改方向         |
| ---- | ------------------ | ---- | ------------ |
| 偶然内聚 | 无关功能放一起            | 最差   | 按职责拆分        |
| 逻辑内聚 | 同类功能放一起，由参数决定执行哪一个 | 较差   | 拆方法、拆类、多态    |
| 时间内聚 | 因同一时间执行而放一起        | 较差   | 拆具体职责，保留流程协调 |
| 过程内聚 | 因执行顺序而放一起          | 一般   | 流程控制与功能职责分离  |
| 通信内聚 | 操作同一数据对象           | 较好   | 按业务职责继续细分    |
| 顺序内聚 | 前一步输出作为后一步输入       | 好    | 保持清晰数据流      |
| 功能内聚 | 全部元素完成同一功能         | 最好   | 保持单一职责       |

## 内聚修改原则

> 提高内聚的核心思想是：**一个模块只围绕一个明确职责组织内部内容**。

常用方法：

1. **按职责拆分模块**：不要把无关功能放在同一个类中；
2. **避免万能类**：例如 `Utils`、`Manager`、`Helper` 中堆积大量无关方法；
3. **减少 type/flag 判断**：逻辑内聚常通过类型参数控制行为，应拆分或多态化；
4. **流程与职责分离**：流程类负责调度，具体类负责具体工作；
5. **围绕核心数据建模**：同一模块内部操作应围绕同一对象或同一目标；
6. **保证数据流清晰**：顺序内聚中要明确每一步输入输出；
7. **遵循单一职责原则**：一个类或模块最好只有一个引起它变化的原因。

> 考试注意：**内聚强调模块内部，耦合强调模块之间；软件设计目标通常是“高内聚、低耦合”**。
