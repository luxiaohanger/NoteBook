## 设计类图

> **设计类图**是在软件设计阶段使用的 UML 类图，用来描述系统中类的结构、属性、方法以及类之间的关系。

#### 设计类图和概念类图的区别

 **概念类图回答“业务中有什么对象”，设计类图回答“程序中要写哪些类”。**

| 对比项 | 概念类图          | 设计类图                                                        |
| --- | ------------- | ----------------------------------------------------------- |
| 阶段  | 需求分析 / 面向对象分析 | 软件设计                                                        |
| 关注点 | 业务概念          | 程序结构                                                        |
| 类   | 学生、课程、订单      | `StudentController`、`StudentService`、`StudentDao`、`Student` |
| 属性  | 重要业务属性        | 属性名、类型、可见性                                                  |
| 方法  | 通常省略          | 需要写主要方法                                                     |
| 关系  | 业务关系          | 调用、依赖、关联、继承、实现                                              |

#### 类的三栏结构

> 设计类图中，一个类通常分为三栏：**类名、属性、操作**。

```text
┌──────────────────────┐
│ 类名                  │
├──────────────────────┤
│ 属性                  │
├──────────────────────┤
│ 操作 / 方法            │
└──────────────────────┘
```

例如：

```text
┌────────────────────────────┐
│ Student                    │
├────────────────────────────┤
│ - id: Integer              │
│ - name: String             │
│ - major: String            │
├────────────────────────────┤
│ + getId(): Integer         │
│ + setName(name: String): void │
└────────────────────────────┘
```

符号含义：

```text
+ public，公有
- private，私有
# protected，受保护
~ package，包内可见
```

> 概念类图可以只写类名和属性；设计类图通常要写**属性类型、方法参数、返回值、可见性**。



#### 设计类图中类的来源

> 设计类图中的类一般来自概念类、用例、分层架构和设计模式。

常见类包括：

```text
实体类 Entity：Student、Course、Order
控制类 Controller：StudentController、OrderController
业务类 Service：StudentService、OrderService
数据访问类 DAO：StudentDao、OrderDao
工具类 Util：DateUtil、StringUtil
接口 Interface：UserRepository、PaymentService
```

#### 类的关系


常见关系：

```text
依赖：使用者 - - -> 被使用者
实现：实现类 - - -▷ 接口
关联：类 ───── 类
聚合：整体 ◇──── 部分
组合：整体 ◆──── 部分
继承：子类 ─────▷ 父类
```


> 类实现接口

> 上层类依赖下层接口

> 关联表示一个类长期持有另一个类的引用

> 依赖关系更弱，表示临时使用，通常指作为参数、局部变量出现，关联作为成员出现

> 组合表示生命周期依附

> 聚合表示生命周期相互独立

> 注意，物理包图通常没有关联依赖的区分，多使用依赖，类图则需要作出区分；概念类图通常只画实体概念类，无需接口

#### 示例：

```
┌──────────────────────┐
│ UserController        │
├──────────────────────┤
│ - userService         │
├──────────────────────┤
│ + getUserById()       │
│ + addUser()           │
└──────────┬───────────┘
           │ 依赖
           ↓
┌──────────────────────┐
│ <<interface>>         │
│ UserService           │
├──────────────────────┤
│ + getUserById()       │
│ + addUser()           │
└──────────▲───────────┘
           │ 实现
┌──────────┴───────────┐
│ UserServiceImpl       │
├──────────────────────┤
│ - userDao             │
├──────────────────────┤
│ + getUserById()       │
│ + addUser()           │
└──────────┬───────────┘
           │ 依赖
           ↓
┌──────────────────────┐
│ <<interface>>         │
│ UserDao               │
├──────────────────────┤
│ + findById()          │
│ + insert()            │
└──────────▲───────────┘
           │ 实现
┌──────────┴───────────┐
│ UserDaoImpl           │
├──────────────────────┤
│ + findById()          │
│ + insert()            │
└──────────┬───────────┘
           │ 依赖
           ↓
┌──────────────────────┐
│ User                  │
├──────────────────────┤
│ - userId              │
│ - username            │
└──────────────────────┘
```
