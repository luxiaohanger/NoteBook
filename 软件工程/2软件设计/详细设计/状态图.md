## 状态图（State Diagram）

状态图用于描述一个对象、系统或业务实体在生命周期中经历的不同状态，以及状态之间因事件触发而发生的转换。

#### 状态图基本元素

| 元素   | 图例                   | 含义           |
| ---- | -------------------- | ------------ |
| 初始状态 | `●`                  | 状态机开始位置      |
| 状态   | 圆角矩形                 | 对象在某一时刻的稳定情况 |
| 转换   | `状态A ──事件/动作──▶ 状态B` | 从一个状态进入另一个状态 |
| 终止状态 | `◎`                  | 状态机结束位置      |
| 判断分支 | 菱形                   | 根据条件进入不同状态   |
| 复合状态 | 大状态框中包含子状态           | 一个状态内部还有细分状态 |

#### 示例：订单状态图

```text
●
│
▼
┌─────────────┐
│   待支付     │
└─────────────┘
   │ pay() [成功] / 生成支付记录
   ▼
┌─────────────┐
│   已支付     │
└─────────────┘
   │ ship() / 发货
   ▼
┌─────────────┐
│   已发货     │
└─────────────┘
   │ confirm() / 确认收货
   ▼
┌─────────────┐
│   已完成     │
└─────────────┘
   │
   ▼
◎
```

异常分支：

```text
待支付 ── cancel() ──▶ 已取消
待支付 ── pay() [失败] ──▶ 支付失败
已发货 ── return() ──▶ 退货中
```
