## 分层物理包图


> **物理包图**用于描述系统实现时的代码包、模块、子系统之间的组织结构和依赖关系。


Web 系统中常见物理包结构如下：

```text
前端
  ↓ REST 请求
controller
  ↓
service
  ↓
dao
  ↓
database
```

各层职责：

```text
controller：接收请求，返回响应
service：处理业务逻辑
dao：访问数据库
entity：表示业务数据对象
database：真正存储数据
```

> `entity` 不是数据库，而是程序中的实体类；数据库才是真正存数据的地方。


#### 依赖关系

> 物理包图中常用虚线箭头表示依赖关系。

```text
A - - -> B
```

含义：

> **A 依赖 B，A 中的代码使用 B 中的类或接口。**

典型依赖：

```text
controller - - -> service
service    - - -> dao
dao        - - -> entity
dao        - - -> database
```

> 依赖箭头指向**被使用的一方**。

最低模板：

> Browser/HTML/CSS/JS -> HTTP/REST -> Controller -> Service -> Dao -> Database
