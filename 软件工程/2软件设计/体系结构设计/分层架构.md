约定：使用分层架构、接口和实现分别在不同包、不同实体同一层次的代码组织在同一包中

## 前后端结构

后端示意图如下：
```
前端
 ↓
Controller
 ↓
Service 接口
 ↑
ServiceImpl
 ↓
Dao 接口
 ↑
DaoImpl
 ↓
数据库
```

如果考虑前端：
```
Client / Frontend
├── View
├── Controller / ViewModel
├── API Service
├── Model / DTO
└── Util
        │
        │ RESTful HTTP / JSON
        ↓
Server / Backend
├── Controller
├── Service
├── ServiceImpl
├── Dao
├── DaoImpl
└── Entity
```

## 接口代码
```java
package com.example.system.service;

public interface UserService {
    boolean register(UserRegisterDTO userRegisterDTO);

    UserVO login(UserLoginDTO userLoginDTO);

    UserVO getUserById(Long userId);

    boolean deleteUser(Long userId);
}
```

```java
package com.example.system.service;

public interface BookService {
    boolean addBook(BookCreateDTO bookCreateDTO);

    boolean updateBook(BookUpdateDTO bookUpdateDTO);

    BookVO getBookById(Long bookId);

    List<BookVO> listBooks();

    boolean deleteBook(Long bookId);
}
```

```java
package com.example.system.dao;

public interface UserDao {
    User findById(Long userId);

    User findByUsername(String username);

    int insert(User user);

    int update(User user);

    int deleteById(Long userId);
}
```

```java
package com.example.system.dao;

public interface BookDao {
    Book findById(Long bookId);

    List<Book> findAll();

    int insert(Book book);

    int update(Book book);

    int deleteById(Long bookId);
}
```
