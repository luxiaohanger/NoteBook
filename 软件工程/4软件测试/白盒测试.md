## 白盒测试

白盒测试根据**代码内部结构、控制流程和逻辑条件**来设计测试用例。它不是只看输入输出，而是要分析代码中有哪些语句、判定、条件和路径需要被执行。

例如有如下代码：

```java
public class DiscountService {

    public int calculateDiscount(boolean isVip, int amount) {
        int discount = 0;

        if (isVip && amount >= 100) {
            discount = 20;
        } else {
            discount = 5;
        }

        return discount;
    }
}
```

这段代码中有一个判定：

```java
if (isVip && amount >= 100)
```

它包含两个基本条件：

```text
isVip
amount >= 100
```

> 白盒测试的核心是“根据代码结构设计测试用例”，而不是只根据需求说明设计测试用例。

> 易错点：白盒测试不等于随便看代码测一下，而是要明确覆盖目标，例如语句覆盖、判定覆盖、条件覆盖、路径覆盖。

---

## 语句覆盖

语句覆盖要求程序中的**每条语句至少执行一次**。

以上面代码为例：

```java
public int calculateDiscount(boolean isVip, int amount) {
    int discount = 0;

    if (isVip && amount >= 100) {
        discount = 20;
    } else {
        discount = 5;
    }

    return discount;
}
```

为了让所有语句都执行，至少需要让 `if` 分支和 `else` 分支都执行一次。

测试用例：

| 用例 | `isVip` | `amount` | 执行分支      | 期望结果 |
| -- | ------: | -------: | --------- | ---: |
| T1 |  `true` |    $100$ | `if` 分支   | $20$ |
| T2 | `false` |     $50$ | `else` 分支 |  $5$ |

测试代码：

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class DiscountServiceTest {

    @Test
    public void testStatementCoverage() {
        DiscountService service = new DiscountService();

        assertEquals(20, service.calculateDiscount(true, 100));
        assertEquals(5, service.calculateDiscount(false, 50));
    }
}
```

> 语句覆盖关注“代码有没有被执行到”。

> 易错点：语句覆盖是较弱的覆盖标准，语句执行过不代表逻辑判断充分测试过。

## 判定覆盖

判定覆盖要求每个判定的结果 `true` 和 `false` 至少各出现一次。

在代码中：

```java
if (isVip && amount >= 100)
```

这个整体判定有两个结果：

| 判定表达式                    | 结果      |
| ------------------------ | ------- |
| `isVip && amount >= 100` | `true`  |
| `isVip && amount >= 100` | `false` |

测试用例：

| 用例 | `isVip` | `amount` | 判定结果    | 期望结果 |
| -- | ------: | -------: | ------- | ---: |
| T1 |  `true` |    $100$ | `true`  | $20$ |
| T2 | `false` |    $100$ | `false` |  $5$ |

测试代码：

```java
@Test
public void testDecisionCoverage() {
    DiscountService service = new DiscountService();

    assertEquals(20, service.calculateDiscount(true, 100));  // 判定为 true
    assertEquals(5, service.calculateDiscount(false, 100));  // 判定为 false
}
```

注意，判定覆盖只要求整个判定表达式取到 `true` 和 `false`，不一定要求每个基本条件都分别取到 `true` 和 `false`。

例如：

| 用例 | `isVip` | `amount >= 100` |    整体判定 |
| -- | ------: | --------------: | ------: |
| T1 |  `true` |          `true` |  `true` |
| T2 | `false` |          `true` | `false` |

这里 `amount >= 100` 一直是 `true`，但判定覆盖已经满足。

> 判定覆盖关注“每个分支方向是否都走过”。

> 易错点：判定覆盖不一定覆盖每个基本条件的真假情况。

---

## 条件覆盖

条件覆盖要求每个判定中的**每个基本条件**都至少取到一次 `true` 和一次 `false`。

对于：

```java
if (isVip && amount >= 100)
```

基本条件有两个：

| 基本条件            | 需要覆盖           |
| --------------- | -------------- |
| `isVip`         | `true`、`false` |
| `amount >= 100` | `true`、`false` |

测试用例：

| 用例 | `isVip` | `amount` | `isVip` | `amount >= 100` |    整体判定 | 期望结果 |
| -- | ------: | -------: | ------: | --------------: | ------: | ---: |
| T1 |  `true` |    $100$ |  `true` |          `true` |  `true` | $20$ |
| T2 | `false` |     $50$ | `false` |         `false` | `false` |  $5$ |

测试代码：

```java
@Test
public void testConditionCoverage() {
    DiscountService service = new DiscountService();

    assertEquals(20, service.calculateDiscount(true, 100));
    assertEquals(5, service.calculateDiscount(false, 50));
}
```

这两个用例使：

```text
isVip: true、false 都出现
amount >= 100: true、false 都出现
```

但是要注意，条件覆盖不一定保证判定覆盖。

例如下面两个用例：

| 用例 | `isVip` | `amount` | `isVip` | `amount >= 100` |    整体判定 |
| -- | ------: | -------: | ------: | --------------: | ------: |
| T1 |  `true` |     $50$ |  `true` |         `false` | `false` |
| T2 | `false` |    $100$ | `false` |          `true` | `false` |

这两个用例满足条件覆盖，但整体判定始终为 `false`，没有覆盖 `if` 分支。

> 条件覆盖关注“每个小条件的真假是否都测过”。

> 易错点：条件覆盖不一定满足判定覆盖，判定覆盖也不一定满足条件覆盖。

## 路径覆盖

路径覆盖要求程序中每条独立路径至少执行一次。它比语句覆盖、判定覆盖和条件覆盖更强，关注的是程序执行路径的组合。

为了更清楚说明路径覆盖，看下面这个有两个独立 `if` 的代码：

```java
public class FeeService {

    public int calculateFee(boolean isMember, boolean hasCoupon) {
        int fee = 100;

        if (isMember) {
            fee = fee - 20;
        }

        if (hasCoupon) {
            fee = fee - 10;
        }

        return fee;
    }
}
```

这段代码有两个判定：

```text
if (isMember)
if (hasCoupon)
```

每个判定都有 `true` 和 `false`，因此路径组合共有：

$$
2^2=4
$$

路径表：

| 路径 | `isMember` | `hasCoupon` | 执行情况        |  期望结果 |
| -- | ---------: | ----------: | ----------- | ----: |
| P1 |     `true` |      `true` | 会员优惠，使用优惠券  |  $70$ |
| P2 |     `true` |     `false` | 会员优惠，不使用优惠券 |  $80$ |
| P3 |    `false` |      `true` | 非会员，使用优惠券   |  $90$ |
| P4 |    `false` |     `false` | 非会员，不使用优惠券  | $100$ |

测试代码：

```java
@Test
public void testPathCoverage() {
    FeeService service = new FeeService();

    assertEquals(70, service.calculateFee(true, true));
    assertEquals(80, service.calculateFee(true, false));
    assertEquals(90, service.calculateFee(false, true));
    assertEquals(100, service.calculateFee(false, false));
}
```

> 路径覆盖关注“程序可能怎么走，每条路线是否都测过”。

> 易错点：路径覆盖在循环、嵌套分支较多时路径数量会迅速增加，实际测试中通常只覆盖主要独立路径。

## 四种白盒覆盖方法对比

| 方法   | 含义                           | 关注对象 | 强度 |
| ---- | ---------------------------- | ---- | -- |
| 语句覆盖 | 每条语句至少执行一次                   | 语句   | 较弱 |
| 判定覆盖 | 每个判定结果 `true` / `false` 至少一次 | 分支方向 | 中等 |
| 条件覆盖 | 每个基本条件 `true` / `false` 至少一次 | 基本条件 | 中等 |
| 路径覆盖 | 每条独立路径至少执行一次                 | 执行路径 | 较强 |

覆盖强度大致可以理解为：

$$
路径覆盖 > 判定覆盖 \quad \text{且} \quad 路径覆盖 > 语句覆盖
$$

但要注意：

$$
判定覆盖 \not\Rightarrow 条件覆盖
$$

$$
条件覆盖 \not\Rightarrow 判定覆盖
$$

> 记忆方式：语句看“行”，判定看“分支”，条件看“小条件”，路径看“整条执行路线”。

> 考试中常见题型是给一段代码，要求设计最少测试用例满足某种覆盖标准。
