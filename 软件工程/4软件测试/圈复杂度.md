## 圈复杂度

圈复杂度也叫**环路复杂度**或 **Cyclomatic Complexity**，用于衡量程序控制流的复杂程度。它反映程序中**线性无关路径**的数量，常用于白盒测试中估算需要多少条基本路径测试用例。

$$
V(G)=E-N+2
$$

其中：

| 符号     | 含义        |
| ------ | --------- |
| $V(G)$ | 圈复杂度      |
| $E$    | 控制流图中的边数  |
| $N$    | 控制流图中的结点数 |

> 圈复杂度越大，说明程序分支越多、路径越复杂，测试和维护难度越高。


---

## 控制流图

控制流图是把程序执行流程表示成图：

| 元素   | 含义                           |
| ---- | ---------------------------- |
| 结点   | 一条或一组顺序执行的语句                 |
| 边    | 控制流转移关系                      |
| 判定结点 | `if`、`while`、`for` 等会产生分支的语句 |

例如代码：

```java
public int fee(boolean isVip, boolean hasCoupon) {
    int price = 100;

    if (isVip) {
        price -= 20;
    }

    if (hasCoupon) {
        price -= 10;
    }

    return price;
}
```

它的控制流程可以理解为：

```text
开始
  |
price = 100
  |
判断 isVip
 /      \
T        F
|        |
price-=20
 \      /
  判断 hasCoupon
 /      \
T        F
|        |
price-=10
 \      /
 return price
```

这里有两个判定：

```text
if (isVip)
if (hasCoupon)
```

所以程序存在多条执行路径。

> 控制流图是计算圈复杂度的基础。

> 易错点：普通顺序语句通常可以合并成一个结点，关键是看分支和循环结构。

## 圈复杂度计算公式

圈复杂度常见计算方法有三种。

### 方法一：边数和结点数

对于一个连通控制流图：

$$
V(G)=E-N+2
$$

如果控制流图有多个连通区域，则一般公式为：

$$
V(G)=E-N+2P
$$

其中 $P$ 是连通区域个数。通常分析单个函数时，$P=1$。

> 考试中最常用的是 $V(G)=E-N+2$。

### 方法二：判定结点数

对于结构化程序：

$$
V(G)=D+1
$$

其中 $D$ 是判定结点个数。

例如：

```java
if (isVip) { ... }

if (hasCoupon) { ... }
```

有两个判定结点：

$$
D=2
$$

所以：

$$
V(G)=D+1=3
$$

> 一个 `if`、`while`、`for` 通常算一个判定结点。

### 方法三：区域数

在控制流图中，圈复杂度等于图中**区域数**，包括最外部区域。

$$
V(G)=区域数
$$

这个方法常用于已经画出控制流图的题目。

> 画图题中可以通过数封闭区域加外部区域得到圈复杂度。

## 代码示例一：顺序结构

顺序结构没有分支。

```java
public int addTax(int price) {
    int tax = price / 10;
    int total = price + tax;
    return total;
}
```

判定结点数：

$$
D=0
$$

圈复杂度：

$$
V(G)=D+1=1
$$

含义是该函数只有一条基本执行路径。

> 顺序代码的圈复杂度通常是 $1$。

> 易错点：圈复杂度最低是 $1$，不是 $0$。

---

## 代码示例二：单个 if

```java
public int discount(boolean isVip) {
    int price = 100;

    if (isVip) {
        price -= 20;
    }

    return price;
}
```

判定结点数：

$$
D=1
$$

圈复杂度：

$$
V(G)=D+1=2
$$

对应两条路径：

| 路径 | `isVip` | 执行情况              |
| -- | ------: | ----------------- |
| P1 |  `true` | 执行 `price -= 20`  |
| P2 | `false` | 不执行 `price -= 20` |

> 一个二分支判定会使圈复杂度增加 $1$。

---

## 代码示例三：两个独立 if

```java
public int fee(boolean isVip, boolean hasCoupon) {
    int price = 100;

    if (isVip) {
        price -= 20;
    }

    if (hasCoupon) {
        price -= 10;
    }

    return price;
}
```

判定结点数：

$$
D=2
$$

圈复杂度：

$$
V(G)=D+1=3
$$

注意：虽然实际路径组合有：

$$
2^2=4
$$

但圈复杂度是：

$$
V(G)=3
$$

因为圈复杂度表示的是**线性无关路径数量**，不是所有可能路径数量。

一组基本路径可以是：

| 基本路径 | `isVip` | `hasCoupon` | 说明           |
| ---- | ------: | ----------: | ------------ |
| P1   | `false` |     `false` | 两个 `if` 都不进入 |
| P2   |  `true` |     `false` | 进入第一个 `if`   |
| P3   | `false` |      `true` | 进入第二个 `if`   |

如果要做完整路径覆盖，还可以再测：

| 路径 | `isVip` | `hasCoupon` |
| -- | ------: | ----------: |
| P4 |  `true` |      `true` |

> 圈复杂度对应基本路径测试的最少独立路径数，不一定等于所有路径数。

> 易错点：两个独立 `if` 的所有路径是 $4$ 条，但圈复杂度是 $3$。

---

## 代码示例四：if-else if-else

```java
public String grade(int score) {
    if (score >= 90) {
        return "A";
    } else if (score >= 60) {
        return "B";
    } else {
        return "C";
    }
}
```

判定结点有两个：

```text
score >= 90
score >= 60
```

所以：

$$
D=2
$$

$$
V(G)=D+1=3
$$

对应基本路径：

| 路径 |   输入 | 执行结果                     |
| -- | ---: | ------------------------ |
| P1 | $95$ | 第一个条件为真，返回 `"A"`         |
| P2 | $70$ | 第一个条件为假，第二个条件为真，返回 `"B"` |
| P3 | $50$ | 两个条件都为假，返回 `"C"`         |

> `else` 本身不增加判定数，真正增加复杂度的是条件判断。

## 代码示例五：循环结构

```java
public int countPositive(int[] nums) {
    int count = 0;

    for (int x : nums) {
        if (x > 0) {
            count++;
        }
    }

    return count;
}
```

这里有两个判定：

| 判定                   | 含义        |
| -------------------- | --------- |
| `for (int x : nums)` | 是否继续循环    |
| `if (x > 0)`         | 当前元素是否为正数 |

所以：

$$
D=2
$$

$$
V(G)=D+1=3
$$

可以设计基本路径测试：

| 路径 | 输入     | 说明             |
| -- | ------ | -------------- |
| P1 | `[]`   | 循环一次也不进入       |
| P2 | `[-1]` | 进入循环，但 `if` 为假 |
| P3 | `[1]`  | 进入循环，且 `if` 为真 |

> 循环结构本身也算判定，因为它要判断是否继续循环。

> 易错点：`for`、`while`、`do-while` 都会增加圈复杂度。

## 代码示例六：复合条件

```java
public boolean canUseCoupon(boolean isVip, int amount) {
    if (isVip && amount >= 100) {
        return true;
    }
    return false;
}
```

从最常见考试角度看，整个 `if` 通常算一个判定：

$$
D=1
$$

所以：

$$
V(G)=D+1=2
$$

但如果使用更严格的工具度量，逻辑运算符 `&&`、`||` 也可能增加复杂度：

```text
isVip && amount >= 100
```

可以看作两个条件共同决定控制流，因为短路求值会影响执行路径。

此时复杂度可能按：

$$
V(G)=2+1=3
$$

理解。

> 课程考试通常按判定结点计算；工程工具可能把 `&&`、`||` 也计入复杂度。

> 易错点：考试时以老师或课件口径为准；如果没有特别说明，通常一个 `if` 算一个判定。

## 圈复杂度与基本路径测试

圈复杂度常用于指导**基本路径测试**。基本路径测试要求选取一组线性无关路径，使这些路径能够覆盖程序的主要控制结构。

基本流程：

1. **画控制流图**：把代码转换成结点和边；
2. **计算圈复杂度**：用 $V(G)=E-N+2$ 或 $V(G)=D+1$；
3. **找基本路径集合**：找出 $V(G)$ 条线性无关路径；
4. **设计测试用例**：为每条基本路径构造输入数据。

> 圈复杂度告诉我们：至少需要多少条独立路径测试用例。

## 圈复杂度总结

| 结构              |        判定数变化 |     圈复杂度变化 |        |       |
| --------------- | -----------: | ---------: | ------ | ----- |
| 顺序结构            |         $+0$ | 不变，通常为 $1$ |        |       |
| `if`            |         $+1$ |       $+1$ |        |       |
| `if-else`       |         $+1$ |       $+1$ |        |       |
| `else if`       |    每个条件 $+1$ |  每个条件 $+1$ |        |       |
| `for` / `while` |         $+1$ |       $+1$ |        |       |
| `case` 分支       | 通常按分支数或判定数计算 |      依课程口径 |        |       |
| `&&` / `        |              |          ` | 工具可能计入 | 依课程口径 |

> 记忆方式：圈复杂度 $\approx$ 判定结点数 $+1$。

> 考试中常考：根据代码求 $V(G)$，根据控制流图用 $E-N+2$ 求 $V(G)$，以及说明圈复杂度与基本路径测试的关系。
