## 用例规约
**用例规约**，也叫**用例描述**，是对某一个用例的详细文字说明。它用于补充用例图中无法表达的细节，比如参与者是谁、执行前需要满足什么条件、正常流程是什么、异常情况怎么处理、执行后系统变成什么状态。

用例图通常只告诉你：

> 用户可以“记录收入”。

但用例规约会详细说明：

> 用户如何记录收入、系统如何校验金额、保存成功后发生什么、输入错误时如何处理。


-------
## 用例规约内容

常见的用例规约模板如下：

| 项目    | 说明                       |
| ----- | ------------------------ |
| 用例名称  | 用例的名称，例如“记录收入”           |
| 用例编号  | 方便管理和追踪，例如 UC-01         |
| 主要参与者 | 直接使用该功能的角色，例如“用户”        |
| 次要参与者 | 辅助参与的系统或角色，例如“数据库”“支付系统” |
| 简要说明  | 这个用例要完成什么目标              |
| 前置条件  | 用例开始前必须满足的条件             |
| 触发条件  | 什么事件会启动这个用例              |
| 基本流程  | 正常情况下用户和系统的交互步骤          |
| 备选流程  | 与主流程不同但仍然正常的流程           |
| 异常流程  | 出错时系统如何处理                |
| 后置条件  | 用例完成后系统应达到的状态            |
| 特殊需求  | 性能、安全、数据格式等非功能或约束需求      |


-------
## 示例：记录收入用例规约

| 项目    | 内容                     |
| ----- | ---------------------- |
| 用例编号  | UC-01                  |
| 用例名称  | 记录收入                   |
| 主要参与者 | 用户                     |
| 简要说明  | 用户录入一条收入信息，系统保存该收入记录   |
| 前置条件  | 用户已登录系统                |
| 触发条件  | 用户点击“新增收入”按钮           |
| 后置条件  | 系统成功保存一条收入记录，并更新收入统计数据 |

**基本流程：**

| 步骤 | 操作                  |
| -- | ------------------- |
| 1  | 用户进入收入管理页面          |
| 2  | 用户点击“新增收入”          |
| 3  | 系统显示收入录入表单          |
| 4  | 用户输入收入金额、收入类别、日期和备注 |
| 5  | 用户点击“保存”            |
| 6  | 系统校验收入金额、类别和日期是否合法  |
| 7  | 系统保存收入记录            |
| 8  | 系统提示用户“收入记录保存成功”    |
| 9  | 系统更新收入列表和统计信息       |

**异常流程：**

| 编号 | 异常情况     | 处理方式             |
| -- | -------- | ---------------- |
| A1 | 收入金额为空   | 系统提示“请输入收入金额”    |
| A2 | 收入金额格式错误 | 系统提示“收入金额应为合法数字” |
| A3 | 收入类别为空   | 系统提示“请选择收入类别”    |
| A4 | 日期为空     | 系统提示“请选择日期”      |
| A5 | 保存失败     | 系统提示“保存失败，请稍后重试” |

**特殊需求：**

| 类型    | 内容                   |
| ----- | -------------------- |
| 数据约束  | 收入金额必须为非负数字，最多保留两位小数 |
| 可用性需求 | 输入错误时应在对应输入框附近显示提示信息 |
| 性能需求  | 保存收入记录的响应时间不超过 2 秒   |
