## 网络的结构

节点：主机节点（host,end system）、交换节点（链路层交换机、网络层路由器、高层负载均衡器设备等）
边： 接入链路（主机节点和交换节点之间access）、骨干链路（数据交换节点之间backbone）

*   **1.网络边缘 edge**：指host,end system，即运行网络应用程序的设备（如个人电脑、服务器、手机）。
*   **2.接入网络 access**：ISP(Internet Service Provider)将端系统连接到边缘路由器（进入网络核心的第一个路由器）的物理网络和介质。主要包括：
    *   **住宅接入**：如DSL（数字用户线）、HFC（混合光纤同轴）。
    *   **企业/机构接入**：通常是以太网（Ethernet）**局域网**。
    *   **无线接入**：如Wi-Fi、蜂窝网络（3G/4G/5G）。
    *   **物理介质**：信号传输的载体，分为**导引型**（如双绞线、同轴电缆、光纤）和**非导引型**（如无线电波）。
*   **3.网络核心 core**：由互联的**路由器**组成的网状网络，负责数据的远距离转发。

## 多路复用（Multiplexing）


#### 1. 电路交换：静态多路复用 (Static Multiplexing)
电路交换对应的“分片”是一种**预分配**机制。

* **实现手段：** 频分 (FDM) 或 时分 (TDM)。
* **逻辑：** 资源在通信开始前就被物理或时间上“切开”了。
* **特性：** * **独占性：** 既然是“分片”，这一片就是你的，别人进不来。
    * **无冲突：** 因为路径和资源是固定的，数据传输时不会发生碰撞。
    * **效率瓶颈：** 如果你占了线却不说话（无数据传输），那片资源就白白浪费了，这就是**静默期（Silent Period）**浪费。


#### 2. 分组交换：统计多路复用 (Statistical Multiplexing)
分组交换对应的“分组”是一种**按需分配**机制。

* **实现手段：** 将数据切成小块（Packets），打上标签（Header），谁有数据谁就发。
* **逻辑：** 链路资源不预先分配给特定用户，而是根据流量的统计特性，谁抢到算谁的。采用存储转发（store and forward）的方式，一个数据包通过一跳（hop）转发到另一个router时，必须被完全存储接受才能转发到下一个router，这样就一次只占用链路中某两个router的全部资源
* **特性：**
    * **高效率：** 只要链路空闲，任何人的分组都能传。不会像电路交换那样出现“空占位子”的情况。
    * **排队与抖动：** 既然是抢占，就可能发生“塞车”（拥塞）。当多个分组同时到达路由器，必须在缓冲区排队，甚至因为溢出而丢包。

## 协议分层与服务模型
**PDU** （protocol data unit）协议数据单元，在不同层面不同

- 应用：message 应用报文
- 传输：segment **tcp报文段**
- 网络层：packet **ip数据报**
- 数据链路层：frame **以太网帧**
- 物理层：bit


**OSI参考模型**：一个七层的理论模型（物理层、数据链路层、网络层、传输层、会话层、表示层、应用层）。

## 网络性能

*   **1.时延**：数据从源到目的地所需的时间，由四部分组成：
    *   **传输时延transmission**：将分组所有比特推入链路所需的时间（分组长度 / 链路带宽）。
    *   **传播时延propagation**：比特在物理链路上传播所需的时间（链路长度 / 传播速度）。
    *   **排队时延queue**：分组在路由器缓冲区中等待处理的时间。
    流量强度 `I = (L*a)/R` 不能超过 1 (a:分组到达队列的平均速率)
    *   **处理时延process**：路由器检查分组首部和决定其去向所需的时间。
*   **2.丢包**：到达路由器的分组可能因为缓冲区已满而被丢弃的比例。上一跳重传、host重传、不重传
*   **3.吞吐量**：接收方接收到数据的速率。端到端吞吐量受路径上**瓶颈链路**的速率限制。（min{R/n}）

## 各层服务特点

| 层次    | 一句话总结                              |
| ----- | ---------------------------------- |
| 应用层   | 面向具体应用，连接性和可靠性取决于所用协议及其底层传输协议。     |
| 传输层   | 提供端到端进程通信，TCP 面向连接且可靠，UDP 无连接且不可靠。 |
| 网络层   | IP 提供主机到主机的无连接、不可靠、尽力而为数据报服务。      |
| 数据链路层 | 提供相邻节点间的单跳传输，是否可靠取决于具体链路协议。        |
| 物理层   | 只负责在介质上传输比特流，不提供可靠性保证。             |

## 层间调用

| 层间交互                   | 调用者          | 被调用对象                    | 典型方法 / 接口                                                        | 传递的数据                       | 作用                        |
| ---------------------- | ------------ | ------------------------ | ---------------------------------------------------------------- | --------------------------- | ------------------------- |
| 应用层 $\rightarrow$ 传输层  | 应用进程         | socket 对象 / TCP / UDP 模块 | `socket()`、`connect()` | 应用数据                        | 应用通过 socket 使用 TCP/UDP 服务 |
| 传输层 $\rightarrow$ 网络层  | TCP / UDP 模块 | IP 模块                    | 内核内部协议栈函数调用                                                      | TCP 报文段 / UDP 数据报           | 加 IP 首部，交给 IP 做主机到主机传输    |
| 网络层 $\rightarrow$ 链路层  | IP 模块        | 链路层模块 / 网卡驱动             | 路由表查询、邻居表 / ARP、驱动接口                                             | IP 数据报                      | 确定出口网卡、下一跳和目的 MAC         |
| 链路层 $\rightarrow$ 物理层  | 网卡驱动         | 网卡硬件                     | 发送队列、DMA、网卡寄存器                                                   | 以太网帧                        | 把帧交给网卡转换成比特流              |
| 物理层 $\rightarrow$ 网络介质 | 网卡硬件         | 双绞线 / 光纤 / 无线信道          | 电信号、光信号、无线信号                                                     | 比特流                         | 在物理介质上传输数据                |
