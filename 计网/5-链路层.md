## 常见链路层协议总览

| 协议                  | 常见场景      | 主要特点                            |
| ------------------- | --------- | ------------------------------- |
| Ethernet / 以太网      | 有线局域网     | 最常见的 LAN 链路层协议，使用 MAC 地址和帧      |
| IEEE 802.11 / Wi-Fi | 无线局域网     | 无线链路层协议，通常有 ACK 和重传             |
| PPP                 | 点对点链路     | 常用于拨号、专线等点对点连接                  |
| HDLC                | 点对点或广域网链路 | 面向比特的链路控制协议                     |
| Frame Relay / 帧中继   | 早期广域网     | 面向虚电路，现已较少使用                    |
| ATM                 | 早期高速广域网   | 使用固定长度信元，不是普通可变长帧               |

## 多路访问协议三种类型

| 类型         | 代表协议                                | 工作原理                                           | 核心特点             |
| ---------- | ----------------------------------- | ---------------------------------------------- | ---------------- |
| **信道划分协议** | TDMA、FDMA、CDMA                      | 把共享信道按**时间、频率或编码**划分成多个互不冲突的资源，每个节点使用自己的资源发送数据 | 基本不冲突，但资源可能被浪费   |
| **随机访问协议** | ALOHA、时隙 ALOHA、CSMA、CSMA/CD、CSMA/CA | 节点有数据就尝试发送；如果发生冲突，则等待一段随机时间后重发                 | 利用率灵活，但可能发生碰撞    |
| **轮流协议**   | 轮询协议、令牌传递协议                         | 节点按照某种顺序轮流获得发送权；拿到发送权的节点才能发送数据                 | 冲突少，公平性较好，但有控制开销 |

> 多路访问协议解决的是：多个节点共享同一条广播信道时，谁在什么时候可以发送数据。

## MAC 地址的表示方式

MAC 地址是链路层地址，通常长度为 **48 bit = 6 字节**，常用 **12 个十六进制数**表示。

`00:1A:2B:3C:4D:5E` 每 1 字节用 2 位十六进制表示

----

## 交换机
交换机层数指的是 OSI 模型的层数
- 二层交换机负责转发 MAC 数据，不更改 MAC 信息
- 三层交换机集成二层交换机和路由器的功能

交换机不改变数据流，只根据入端口学习 MAC 和端口映射，并选择转发、丢弃、泛洪，也就是交换机只能决定发送的形式

传统总线型以太网属于广播信道，需要 CSMA/CD。

交换机连接主机时，主机到交换机之间通常是点对点链路，全双工通信下不需要 CSMA/CD。

## 默认网关
网络层和链路层默认**子网和二层网络一致**,这样网络层路由表的匹配规则就变成了：
子网/二层网络内 ip ： APR 直达
其他网络 ip ： 下一跳为默认网关
```
主机要发送 IP 数据报
        ↓
查路由表，对目的 IP 做最长前缀匹配
        ↓
如果匹配直连路由：
    下一跳 = 目的主机本身
    ARP 目的 IP
        ↓
如果匹配非直连路由：（特殊情况）
    下一跳 = 路由项里的 via 地址
    ARP 下一跳 IP
        ↓
如果没有其他匹配：
    匹配 0.0.0.0/0 默认路由
    ARP 默认网关 IP
```

## 广播

广播常用于发送方不知道目标地址，或者需要通知同一局域网内所有设备的情况。

常见例子：

1. **ARP 请求**：主机知道目标 IP，但不知道目标 MAC，于是广播询问：

$$
\text{谁拥有这个 IP 地址？请告诉我你的 MAC 地址}
$$

2. **DHCP Discover**：主机刚接入网络时还没有 IP 地址，于是广播寻找 DHCP 服务器；

3. **局域网服务发现**：某些协议会用广播发现同一局域网内的设备或服务。

---

## 泛洪

**泛洪**是指网络设备收到一个帧或分组后，把它从除入口端口以外的多个端口都转发出去。

以交换机为例，如果交换机收到一个以太网帧，但**不知道目的 MAC 地址对应哪个端口**，它就会执行泛洪：

1. **收到帧**：交换机从某个端口收到以太网帧；
2. **学习源地址**：记录源 MAC 地址和入端口的映射；
3. **查找目的地址**：查看 MAC 地址表中是否有目的 MAC 对应端口；
4. **未知则泛洪**：如果没有表项，就从除入口端口外的所有端口转发；
5. **等待学习**：目标主机回复后，交换机学习目标 MAC 与端口的映射。

## 广播域

> **广播域**是一次广播能够到达的网络范围。

在传统以太网中：

$$
\text{一个 VLAN 通常对应一个广播域}
$$

路由器或三层交换机的三层接口通常会隔离广播域。


> 交换机默认会扩展广播域，路由器默认会隔离广播域。VLAN 可以把一台交换机上的端口划分进不同广播域。

## CSMA/CD 最短帧长

CSMA/CD 的**最短帧长**是为了保证：发送站在发送一帧的过程中，如果发生冲突，必须还能检测到冲突。

在以太网 CSMA/CD 中，最短帧长满足：

$$
最短帧长 \ge 2\times 最大传播时延 \times 数据传输速率
$$

也可以写成：

$$
L_{\min}=2\tau R
$$

其中：

| 符号         | 含义              |
| ---------- | --------------- |
| $L_{\min}$ | 最短帧长，单位通常是 bit  |
| $\tau$     | 单程最大传播时延        |
| $2\tau$    | 往返传播时延，也叫争用期    |
| $R$        | 数据传输速率，单位 bit/s |

> 帧不能太短，否则发送完了还没来得及发现远端冲突，就误以为发送成功了。

> CSMA/CD 的关键不是“避免冲突”，而是“冲突发生时能检测出来”。因此最短帧长至少要覆盖一个往返传播时延。

> 一般题目会给以比特时间为单位的最大传播时延，已经把速率考虑进去了，此时直接乘二倍即可

## 数据中心网络等级拓扑总览

| 层次   | 位置       | 连接对象                                                      | 主要作用                   | 特点 / 注意点             |
| ---- | -------- | --------------------------------------------------------- | ---------------------- | -------------------- |
| 接入层  | 最靠近服务器   | 服务器、机架内交换机、ToR 交换机                                        | 接入服务器流量，把服务器连接到数据中心网络  | 是服务器进入网络的第一跳         |
| 汇聚层  | 接入层上方    | 多个接入交换机                                                   | 汇聚多个机架或区域的流量，并向核心层转发   | 可承担访问控制、负载均衡、策略管理等功能 |
| 核心层  | 网络最上层    | 多个汇聚交换机、外部网络                                              | 提供高速骨干转发，连接外部网络或其他数据中心 | 重点是高速、可靠、低时延转发       |


> 服务器 $\rightarrow$ 接入层 $\rightarrow$ 汇聚层 $\rightarrow$ 核心层
