## Context Switch（上下文切换）

上下文切换是操作系统把 CPU 从一个正在运行的执行流切换到另一个执行流的过程。这里的执行流可以是**进程**，也可以是**线程**。

| 切换类型      | 切换对象             | 地址空间是否变化 | 开销大小 | 典型场景            |
| --------- | ---------------- | -------- | ---- | --------------- |
| 线程上下文切换   | 同一进程内线程之间切换      | 通常不变     | 较小   | 多线程程序调度         |
| 进程上下文切换   | 不同进程之间切换         | 通常变化     | 较大   | 时间片用完、阻塞、唤醒其他进程 |
| 内核态/用户态切换 | 同一进程在用户态和内核态之间切换 | 不变       | 较小   | 系统调用、中断、异常      |

上下文切换的核心目的，是让多个进程或线程**共享一个或多个 CPU**，从而实现并发执行。

$$
CPU \rightarrow 保存当前上下文 \rightarrow 选择新任务 \rightarrow 恢复新上下文 \rightarrow CPU
$$

> 通俗地说，上下文切换就是操作系统先“存档”当前程序的运行状态，再“读档”另一个程序继续运行。


---

## 触发上下文切换的常见原因

| 触发原因     | 是否一定发生上下文切换 | 说明                        |
| -------- | ----------: | ------------------------- |
| 时间片用完    |         通常会 | 抢占式调度中，时钟中断触发调度           |
| 进程主动阻塞   |           会 | 例如等待 I/O、等待锁、调用 `sleep()` |
| 进程退出     |           会 | 当前进程不能继续运行，必须调度其他进程       |
| 高优先级任务就绪 |         可能会 | 抢占式系统可能立即切换               |
| 系统调用     |         不一定 | 系统调用会发生用户态到内核态切换，但不一定换进程  |
| 中断/异常    |         不一定 | 可能只是处理完后返回原进程             |

## 进程上下文切换与线程上下文切换

| 对比项    | 进程上下文切换     | 线程上下文切换   |
| ------ | ----------- | --------- |
| 切换对象   | 不同进程        | 同一进程内不同线程 |
| 地址空间   | 通常不同        | 同一进程内通常相同 |
| 页表切换   | 通常需要        | 通常不需要     |
| TLB 影响 | 可能导致 TLB 失效 | 影响较小      |
| 资源共享   | 进程间资源隔离     | 线程共享进程资源  |
| 开销     | 较大          | 较小        |
