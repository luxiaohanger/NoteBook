## 磁盘基本原理总览

| 考点        | 核心含义                    | 关键记忆                            |
| --------- | ----------------------- | ------------------------------- |
| 磁盘组成      | 用旋转盘片、磁头、磁道、扇区组织数据      | 磁盘是机械设备，慢主要慢在“移动”和“等待旋转”        |
| 数据寻址      | 早期按“盘面、磁道、扇区”寻址，现代用 LBA | OS 看到的是线性块号，不直接关心真实几何结构         |
| 一次读写过程    | 寻道、等待旋转、传输数据            | 访问时间 = 寻道时间 + 旋转延迟 + 传输时间       |
| 磁盘优化      | 磁道偏移、多区域磁盘、缓存           | 顺序访问远快于随机访问                     |
| 磁盘调度      | 调整请求顺序，减少磁头移动           | FCFS 公平但慢，SSTF 快但可能饥饿，SCAN 类似电梯 |
| HDD 与 SSD | HDD 有机械延迟，SSD 无寻道和旋转延迟  | SSD 快，但有擦写寿命和 FTL 问题            |

## 磁盘的物理结构

磁盘把数据磁性地存储在旋转盘片表面的薄金属膜上。盘片通常由玻璃、陶瓷或铝等材料构成，表面涂有可磁化材料。磁头负责读写数据：读取时感知磁场，写入时改变磁场。

| 结构                | 含义                          |
| ----------------- | --------------------------- |
| 盘片 Platter        | 存储数据的圆盘                     |
| 盘面 Surface        | 盘片的一个表面                     |
| 磁道 Track          | 盘面上一圈同心圆轨道                  |
| 扇区 Sector / Block | 磁盘读写的基本单位，一般为 512 字节        |
| 柱面 Cylinder       | 不同盘面上相同磁道编号的一组磁道            |
| 磁头 Head           | 负责读取或写入磁性信息                 |
| 转轴 Spindle        | 带动盘片旋转，常见转速为 4200–15000 RPM |
| 磁臂 Arm assembly   | 移动磁头到指定磁道                   |

> 磁盘可以理解为“高速旋转的多层唱片”，磁头移动到指定位置后读写某一小块数据。

> 扇区读写通常被看作原子操作；考试中不要把“字节”当作磁盘硬件读写的最小单位。

## 磁盘数据寻址

早期磁盘使用物理地址定位数据：

$$
\text{Address}=(\text{drive},\text{surface},\text{track},\text{sector})
$$

现代磁盘通常使用逻辑块地址 LBA，也就是 Logical Block Address。操作系统看到的是从 $0$ 到 $N-1$ 的线性块号，磁盘控制器内部再把 LBA 映射到具体物理位置。

$$
\text{LBA}=0,1,2,\dots,N-1
$$

> LBA 就是把复杂的“盘面、磁道、扇区”包装成连续数组下标。

> 现代操作系统通常不直接依赖磁盘真实几何结构，但仍然可以假设相邻扇区号在物理上大概率相邻，因此顺序访问更快。

## 一次磁盘读写过程

一次磁盘读写大致包括以下步骤：

1. **定位磁道**：磁头移动到目标磁道，这一步叫寻道；
2. **等待稳定**：磁头到位后需要稳定；
3. **选择磁头**：启用对应盘面的磁头；
4. **等待扇区旋转到磁头下方**：这一步叫旋转延迟；
5. **读写扇区**：真正传输数据。

磁盘访问时间公式为：

$$
T_{I/O}=T_{seek}+T_{rotation}+T_{transfer}
$$

磁盘传输速率为：

$$
R_{I/O}=\frac{Size_{transfer}}{T_{I/O}}
$$

其中：

| 项              | 含义                   |
| -------------- | -------------------- |
| $T_{seek}$     | 寻道时间，磁头移动到目标磁道的时间    |
| $T_{rotation}$ | 旋转延迟，等待目标扇区转到磁头下方的时间 |
| $T_{transfer}$ | 传输时间，真正读写数据的时间       |

> 磁盘读写不是“CPU 直接拿数据”，而是先移动磁头，再等盘片转到正确位置，最后才传输数据。

> 随机访问慢主要是因为每次都可能产生新的寻道时间和旋转延迟；顺序访问快是因为后续扇区通常很快连续到达磁头下方。

## 磁盘读写性能特点

在机械硬盘 HDD 中，主要瓶颈通常不是传输本身，而是：

$$
T_{seek}+T_{rotation}
$$

也就是说，**随机小块读写很慢，顺序大块读写较快**。

例如顺序传输 $100\text{ MB}$ 数据时，如果平均寻道 $4\text{ ms}$，平均旋转 $2\text{ ms}$，传输时间 $0.8\text{ s}$，则总时间为：

$$
T_{I/O}=0.004+0.002+0.8=0.806\text{ s}
$$

传输速率为：

$$
R_{I/O}=\frac{100\text{ MB}}{0.806\text{ s}}\approx124\text{ MB/s}
$$

> 大文件顺序读写时，寻道和旋转只占很小一部分；大量小文件随机读写时，机械动作开销会被反复放大。

> 做题时看清楚是“顺序传输”还是“随机访问”，二者性能差异很大。

## 磁道偏移

磁道偏移 Track Skew 是指相邻磁道的扇区编号不是完全对齐排列，而是故意错开一点。

原因是：当磁头从一个磁道移动到下一个磁道时需要时间。如果相邻磁道的下一个扇区正好对齐，那么磁头移动完成时，目标扇区可能已经转过去了，必须再等一整圈。

> 磁道偏移就是提前错开编号，让磁头换道后刚好赶上下一个要读的扇区。

> 它是为了优化顺序读，不是为了改变磁盘容量。

## 多区域磁盘

多区域 Multi-Zoned 磁盘是指外圈磁道比内圈磁道包含更多扇区。

原因是外圈周长更长，可以放下更多扇区。磁盘被划分为多个区域，每个区域由一组连续磁道构成，同一区域内每条磁道扇区数相同。

| 位置   | 特点        |
| ---- | --------- |
| 外圈磁道 | 周长更长，扇区更多 |
| 内圈磁道 | 周长更短，扇区较少 |

> 外圈地方大，所以能放更多扇区。

> 现代磁盘通过 LBA 隐藏这些细节，操作系统一般不需要直接知道每条磁道到底有多少扇区。

## 磁盘缓存

磁盘内部通常有小容量缓存，也叫磁道缓冲区，用于暂存读写数据。读取某个扇区时，磁盘可能顺便把同一磁道上的多个扇区读入缓存。写入时常见策略有两种：

| 写策略              | 含义                   | 特点          |
| ---------------- | -------------------- | ----------- |
| Write Through 直写 | 数据真正写入磁盘后才报告完成       | 更可靠，但较慢     |
| Write Back 写回    | 数据进入缓存后就报告完成，之后再写入磁盘 | 更快，但断电时风险更高 |

> 缓存利用的是局部性：你读了一个扇区，很可能马上还会读附近扇区。

> 写回缓存提高性能，但不能简单等同于“数据已经安全落盘”。

## 磁盘调度

由于磁盘 I/O 成本较高，操作系统可以调整磁盘请求的处理顺序，目标是减少磁头移动距离，提高吞吐量。

| 算法     | 核心思想                | 优点          | 缺点               |
| ------ | ------------------- | ----------- | ---------------- |
| FCFS   | 按请求到达顺序处理           | 公平，顺序符合应用预期 | 可能导致大量随机寻道       |
| SSTF   | 优先处理离当前磁头最近的请求      | 平均寻道距离短     | 远处请求可能饥饿         |
| SCAN   | 磁头像电梯一样单方向扫描，到端点后反向 | 减少来回抖动，较公平  | 边缘请求等待可能较久       |
| F-SCAN | 扫描时冻结当前队列，新请求放到下一轮  | 缓解饥饿        | 实现更复杂            |
| C-SCAN | 只按一个方向扫描，到端点后回到起点   | 等待时间更均匀     | 回跳过程可能不服务请求      |
| SPTF   | 同时考虑寻道时间和旋转延迟       | 理论上更精确      | OS 通常不知道磁头具体旋转位置 |

> 磁盘调度的本质是：不要让磁头来回乱跑。

> SSTF 不等于最公平；SCAN 不一定总是最短移动距离，但更稳定。

## HDD 与 SSD 对比

| 项       | HDD 机械硬盘       | SSD 固态硬盘          |
| ------- | -------------- | ----------------- |
| 存储介质    | 磁性盘片           | NAND 闪存           |
| 是否有机械部件 | 有磁头、盘片、转轴      | 没有机械运动部件          |
| 主要延迟    | 寻道时间、旋转延迟、传输时间 | 主要是闪存访问和控制器管理开销   |
| 随机访问    | 较慢             | 很快                |
| 功耗      | 相对较高           | 较低                |
| 特殊问题    | 机械延迟明显         | 擦写寿命、FTL、垃圾回收、写放大 |

SSD 没有旋转和寻道电机，因此消除了机械硬盘中的寻道时间和旋转延迟，访问时间通常远低于 HDD。SSD 使用 NAND 闪存，内部以页面和块组织数据，并通过 FTL 维护逻辑块到物理页面的映射。

> HDD 慢在机械运动，SSD 快在没有机械运动。

> SSD 虽然快，但不是“直接覆盖写入”；由于闪存擦除成本高，常用写时复制和垃圾回收。
